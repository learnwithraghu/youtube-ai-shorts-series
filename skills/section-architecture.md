# Skill: Section Architecture

## Purpose
Use this skill when expanding or revising course sections.

## Inputs
- course goal
- target audience
- current section guide
- any constraints on length or tooling

## Output
A section-level document that includes:
- section title
- what it teaches
- why it matters
- key topics
- output/artifact
- transitions to the next section

## Rules
- Start with the production problem
- Keep sections sequential and cumulative
- Avoid jumping into code too early
- Ensure every section supports later code, slides, or exercises
- Keep the shipping support assistant context consistent
