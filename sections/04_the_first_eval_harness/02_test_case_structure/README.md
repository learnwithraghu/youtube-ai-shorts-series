# 02 — Test Case Structure

## Purpose
Show how to represent a strong eval example.

## What this chapter teaches
- how to describe a scenario clearly
- what input, context, and expected behavior should look like
- how a test case becomes reusable evaluation data

## Topics covered
- user input
- supporting context
- expected response behavior
- references or gold standards
- metadata for scenario classification

## Talking points
- a good eval case is specific enough to judge
- context matters as much as the prompt
- test case design shapes what the metrics can tell us

## Output
A reusable test case template.
