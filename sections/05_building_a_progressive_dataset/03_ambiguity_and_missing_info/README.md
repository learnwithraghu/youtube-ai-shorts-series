# 03 — Ambiguity and Missing Info

## Purpose
Teach how to evaluate uncertain or incomplete user requests.

## What this chapter teaches
- how vague requests behave in production
- how missing data affects answers
- when the assistant should ask follow-up questions

## Topics covered
- ambiguous user messages
- missing shipment or order information
- follow-up question behavior
- uncertainty handling in support flows

## Talking points
- real users are often incomplete or unclear
- good assistants know when not to guess
- ambiguity is a common source of failures

## Output
A set of ambiguity-focused eval cases.
