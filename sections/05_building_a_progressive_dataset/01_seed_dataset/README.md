# 01 — Seed Dataset

## Purpose
Start with a small but representative set of evaluation cases.

## What this chapter teaches
- why a small dataset is enough to begin
- how to choose strong seed cases
- why the core workflows should be covered first

## Topics covered
- minimum viable eval set
- selecting representative shipping support scenarios
- balancing usefulness with simplicity

## Talking points
- the first dataset is a foundation, not the final benchmark
- good seed cases expose the main risks quickly
- starting small keeps the system understandable

## Output
A minimal but meaningful starting eval set.
