# 02 — The Shipping Support Assistant Context

## Purpose
Introduce the concrete product we will use throughout the course.

## What this chapter teaches
- What the shipping support assistant is responsible for
- What support workflows it must handle
- How product scope creates evaluation scope

## Core workflows
- shipment tracking
- delay explanation
- refund or claim guidance
- customs issue guidance
- human handoff when the assistant cannot safely answer

## Talking points
- This is not a generic chatbot
- It is a support tool tied to operational data
- Different customer questions need different behaviors
- The assistant must know when to answer and when to escalate

## Suggested flow
1. Describe the company and users
2. Show the main support requests
3. Explain the assistant’s job boundaries
4. Transition into why each workflow can fail differently

## Output
A clear product context that the rest of the course can reference.
