# 04 — Evals as a Product Requirement

## Purpose
Explain why evaluation is part of shipping a reliable LLM product.

## What this chapter teaches
- Evals are how we detect failures before users do
- Evals support debugging, regression testing, and release decisions
- Without evals, improvement becomes guesswork

## Talking points
- We need a way to compare versions consistently
- We need a way to measure groundedness and safety, not just fluency
- Evals turn subjective quality into an operational workflow

## Suggested flow
1. Revisit the failure examples
2. Ask how a team would catch them systematically
3. Introduce evals as the answer
4. Set up the move into building the actual eval harness later

## Output
Learners should understand why the rest of the course is structured around evals.
