# 02 — Tools, Data, and Boundaries

## Purpose
Show what information the assistant can trust and what actions it can take.

## What this chapter teaches
- what backend data the assistant depends on
- why tools matter for grounded responses
- how access boundaries shape safety
- when the assistant should stop and hand off

## Topics covered
- shipment records
- order and customer metadata
- policy lookup data
- escalation and ticket creation
- tool access limits
- data freshness and trust boundaries

## Talking points
- the assistant should not invent data
- tools define what can be verified
- if the data is missing, the assistant should say so

## Output
A simple dependency model for the assistant’s behavior.
