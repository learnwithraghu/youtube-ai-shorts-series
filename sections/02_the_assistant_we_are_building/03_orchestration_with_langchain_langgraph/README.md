# 03 — Orchestration with LangChain/LangGraph

## Purpose
Explain how the assistant routes requests and decides what to do next.

## What this chapter teaches
- how intent routing works
- how tool calls fit into the response flow
- the difference between stateful and stateless behavior
- why orchestration is part of reliability

## Topics covered
- request flow from user input to final answer
- intent classification or routing
- tool invocation patterns
- chaining reasoning with tool output
- when LangGraph-style stateful orchestration helps

## Talking points
- the assistant is a workflow, not just a prompt
- routing mistakes can create wrong answers
- structured orchestration makes behavior easier to test

## Output
A buildable architecture for the support assistant.
