# 01 — Scope the Support Problem

## Purpose
Define exactly what the assistant is responsible for and where its limits are.

## What this chapter teaches
- how to scope the assistant around real user needs
- which support requests are in scope
- which requests should be escalated or rejected
- why tighter scope produces better reliability

## Topics covered
- shipping support as the central product use case
- common customer intents:
  - shipment tracking
  - delivery delays
  - refund or claim questions
  - customs questions
  - escalation requests
- in-scope vs out-of-scope behavior
- why generic assistants are harder to trust

## Talking points
- a support assistant should solve a narrow set of jobs well
- unclear scope leads to unclear behavior
- evals become possible only after the product boundaries are defined

## Output
A clean scope definition for the assistant.
