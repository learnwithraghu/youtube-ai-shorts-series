# 03 — Latency and Tool Metadata

## Purpose
Capture operational details that reveal performance and tool reliability issues.

## What this chapter teaches
- why latency belongs in the quality story
- how tool metadata helps explain failures
- how slow or missing tools affect the assistant’s behavior

## Topics covered
- response latency
- tool response time
- tool error states
- missing or stale tool results
- operational signals that indicate user-facing risk

## Talking points
- a fast wrong answer is still wrong
- a slow tool can degrade the user experience even if the text response looks fine
- metadata helps distinguish model issues from tool issues

## Output
A richer observability layer for the support assistant.
