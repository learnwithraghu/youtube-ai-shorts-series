# 04 — Failure Capture Pipeline

## Purpose
Turn real assistant failures into reusable evaluation examples.

## What this chapter teaches
- how to categorize failures
- how to capture replayable examples
- how to convert incidents into eval cases

## Topics covered
- failure tagging and categorization
- replayable traces
- example selection for evals
- building a failure bank
- using production-like issues to improve quality coverage

## Talking points
- the best eval cases come from real failures
- if we do not capture failures, we repeat them
- failure capture turns debugging into long-term leverage

## Output
A pipeline from production issue to eval-ready example.
