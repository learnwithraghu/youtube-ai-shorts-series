# 02 — Trace and Event Structure

## Purpose
Show how to organize logs into a trace that is easy to inspect and replay.

## What this chapter teaches
- how to connect steps inside one request
- how to make events readable and debuggable
- why trace shape matters for later eval workflows

## Topics covered
- request IDs and conversation IDs
- event ordering
- input, tool, and output events
- step-by-step trace design
- replay-friendly structure

## Talking points
- a trace should tell the story of a request
- structure matters more than volume
- clean event modeling reduces debugging time

## Output
A trace model that supports debugging and replay.
