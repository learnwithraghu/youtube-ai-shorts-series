# 01 — What to Log

## Purpose
Identify the minimum set of data needed to debug the assistant and evaluate it later.

## What this chapter teaches
- which interaction fields matter most
- why the raw user message alone is not enough
- how logs support debugging, replay, and evaluation

## Topics covered
- user input
- assistant output
- tool calls and tool results
- timestamps
- request metadata
- session or conversation identifiers

## Talking points
- without logs, failures become anecdotes
- logs let us reconstruct what happened
- the goal is not to log everything; it is to log what helps us reason about behavior

## Output
A practical logging checklist for the assistant.
