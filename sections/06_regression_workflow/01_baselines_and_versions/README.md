# 01 — Baselines and Versions

## Purpose
Define the reference point that all future changes will be compared against.

## What this chapter teaches
- what a baseline is
- why version tracking matters
- how to keep comparisons consistent

## Topics covered
- baseline builds
- model versions
- prompt versions
- tool versions and behavior consistency

## Talking points
- without a baseline, change is hard to judge
- versioning makes quality comparisons meaningful
- regression work depends on stable reference points

## Output
A stable comparison target.
